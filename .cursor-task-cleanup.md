# Task: Full ClawVault Repo Cleanup — Plugin-First Architecture

## Vision

ClawVault is an **OpenClaw memory plugin** that also ships a CLI. The repo should reflect this cleanly.

## Phase 1: Plugin — Eliminate CLI Shelling

### vault-context-injector.ts — CRITICAL

The pre-prompter currently shells out to `clawvault context` and `clawvault search` CLI via `runClawvault()` (which calls `execFileSync`). This causes SQLite locks, timeouts, and fragility.

**Refactor to use `ClawVault.find()` in-process** — the same path that `memory_search` tool uses successfully in `memory-manager.ts`.

Specifically in `src/plugin/vault-context-injector.ts`:
1. `fetchMemoryContextEntries()` — Replace `runClawvault(["context", ...])` with `ClawVault.find()` call
2. `fetchSessionRecapEntries()` — Replace `runClawvault(["session-recap", ...])` with in-process session recap logic
3. Remove the fallback `runClawvault(["search", ...])` path — `ClawVault.find()` IS the search

Reference working pattern in `src/plugin/memory-manager.ts` lines 155-175 (the `search()` method).

### session-lifecycle.ts — Remove CLI calls

`src/plugin/hooks/session-lifecycle.ts` shells out for:
- `runClawvault(["recover", ...])` — startup recovery
- `runClawvault(["checkpoint", ...])` — auto-checkpoint
- `runObserverCron(...)` — observer on reset

These should use library calls where possible. For observer (which is complex), keep the CLI call but make it non-blocking (fire-and-forget with spawn, not execFileSync).

### clawvault-cli.ts — Minimize but keep

This file has helpers for shelling out to the CLI. After the above refactors, most of its functions become dead code. Remove dead functions, keep only what's still used (e.g., observer cron, any remaining CLI-only features).
